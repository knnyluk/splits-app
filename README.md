# Splits Quest

Flutter implementation of the [Splits Quest](https://api.anthropic.com/v1/design/h/83ESD-p81zWnaEB2tgDQtg?open_file=index.html) Android design — a gamified flexibility tracker. Train your splits, earn gems, decorate your room, and dress up your avatar.

## Screens

- `Customize` — Avatar customization (body / hair / skin)
- `Room` — Home base with placed decor and a meditating avatar; opens a "place items" panel
- `Progress` — Train CTAs + per-side progress chart + goals
- `Store` — Buy decor with gems, items unlock by level
- `Inventory` — Owned items with "PLACED" badges

## Run

This repo contains the `lib/` and `pubspec.yaml` only — Android scaffolding is generated by Flutter.

```bash
# install Flutter (one-time)
brew install --cask flutter
flutter doctor

# from this directory, fill in Android scaffolding around existing lib/
flutter create --platforms=android .

# install deps and run on a connected device / emulator
flutter pub get
flutter run
```

Boot an emulator from Android Studio's AVD Manager if you don't have a USB device. `flutter devices` lists what's available.

## Tap-through

- Customize → Finished → Room
- Room → Start → Progress
- Room → Go to Store → Store
- Room → Inventory → Inventory
- Room → right chevron → store panel opens, avatar shrinks to bust pose
- Room → left chevron → panel closes
- Room → tap avatar → re-enter Customize (back to return)
- Progress / Store / Inventory → back arrow → Room

## Design provenance

The source design lives in `/tmp/design-extract/test-flutter-app/project/` — open `index.html` in a browser for a side-by-side reference. The palette, type, and avatar/store-item paths are all reproduced from `styles.css`, `shared.jsx`, and `screen-*.jsx`.

The design's Cover artboard, DEBUG ribbons, and the React `DesignCanvas`/`AndroidFrame` wrappers are intentionally dropped — they're design-tool chrome rather than app UI.

The CV / camera split-measurement feature is intentionally out of scope (matches the original design intent in the chat transcript).
